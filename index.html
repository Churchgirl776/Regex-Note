QUESTION 1: How innerHTML, textContent, and innerText Work

innerHTML: Gets or sets the HTML content inside an element. It includes tags and can be used to insert HTML structure, not just text.

textContent: Gets or sets only the text content of an element and all its children. It does not include any HTML tags and includes hidden text.

innerText: Similar to textContent, but it respects the rendered appearance — it excludes hidden text (like elements styled with display: none).

QUESTION 2: Why Event Handlers Don't Always Need Named Functions

When handling events, we often use anonymous functions — functions without names — because they're usually short and used only once. This avoids cluttering the global scope with unnecessary function names.

Example: element.addEventListener('click', function() { alert('Clicked!'); });

However, using named functions can improve readability and make it easier to remove listeners later with removeEventListener.
